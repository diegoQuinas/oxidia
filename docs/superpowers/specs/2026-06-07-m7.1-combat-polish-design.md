# M7.1 — Combat Polish — Oxidia

> Three TFS-faithful polish items found during M7 live testing: a correct
> **death → logout flow** (no in-world auto-respawn), the **protection-zone
> client badge**, and a **blood-hit effect** fix. Read alongside the M7 design
> (`docs/superpowers/specs/2026-06-07-m7-combat-design.md`).

## Scope

In scope (M7.1):

1. **Death flow rework** — on death, send the relogin/death window, remove the
   player from the world, and end the session. Respawn at the temple happens on
   the next login, not via an in-world teleport.
2. **Protection-zone badge** — show the PZ icon (`ICON_PIGEON`) in the client
   status area while the player stands on a protection-zone tile.
3. **Blood-hit effect** — the `DRAWBLOOD` magic effect is already broadcast on
   every hit; fix its wire value so the client actually renders it.

Explicitly **deferred** (overlap with later milestones, confirmed with the
roadmap owner):

- **Floor blood splat** (`ITEM_SMALLSPLASH`/`FLUID_BLOOD` + decay, TFS
  `game.cpp:3878-3914`) → **M9** (needs the ground-item + decay engine).
- **Corpse body** (`Player::getCorpse`, TFS `player.cpp:2129`) → **M13** (loot &
  corpses; needs items/containers).
- **Skull / PK system** (white/red/black, frags) → **M8.1** (needs M8 to persist
  skull state + frag timestamps).

## 1. Death flow rework

### TFS reference

`Player::death` → `sendReLoginWindow(unfairFightReduction)` (`player.cpp:2070`)
then `g_game.removeCreature(this)` (`player.cpp:2197`, `game.cpp:568`). Death
**removes the creature from the world**; the player is sent the relogin window
and returns to character select. The respawn position is the town temple,
applied on the next login.

### Current behaviour (wrong)

`do_death` (`world/src/game.rs`) sends `0x28`, removes the victim from the death
tile, then **auto-teleports** it: `free_spawn_near(temple)`, re-adds the creature
in-world, restores HP, re-renders bystanders, and prunes the victim's known-set
(the M7-review W1/W2 logic). The player "respawns" instantly without leaving the
game — the bug the roadmap owner observed live.

### New behaviour

`do_death(victim_id)`:

1. Push `combat_packets::death_window(0)` (`0x28`) to the victim.
2. Broadcast the **id-form** remove (`walk::remove_creature_by_id(victim_id)`) to
   the death-tile spectators and drop the victim from each spectator's known-set
   (already correct from the prior fix).
3. **Remove the victim from the `players` map.** This drops the victim's
   `push_tx`, which the session writer observes as a closed channel and tears the
   session down (writer ends → reader's `select!` ends → socket closes). The
   client shows the death window, then returns to character select.
4. On the next login, `login()` runs fresh and uses `free_spawn` (the temple),
   spawning the player at full HP. No death-specific respawn code is needed.

### What this removes

- `free_spawn_near` and the in-`do_death` re-add / HP-restore / teleport-puff.
- The W1/W2 respawn-render and known-set-prune logic **and their tests**
  (`respawn_map_desc_includes_bystander_near_temple`,
  `respawn_prunes_known_set_of_out_of_view_creatures`) — moot once death is a
  logout. The `death_remove_uses_id_form_for_coocc_safety` test stays.

### Risk (verify live)

The writer must flush the `0x28` **before** the session closes. The order is:
push `0x28`, then drop `push_tx`. mpsc delivers buffered messages before
signalling close, so the writer should drain and send the `0x28`, then see the
channel closed and end. Confirm the death window renders live before the socket
drops.

### Future coupling

When **M8** restores a player's last saved position on login, death must reset
the saved position to the temple (so a relog after death lands at the temple, not
the death tile). Until then, login always spawns at the temple, so this is a
no-op for M7.1 — noted for M8/M8.1.

## 2. Protection-zone badge

### TFS reference

`Player::getClientIcons` (`player.cpp:381`): `if (isInProtectionZone()) icons |=
ICON_PIGEON;` and it clears `ICON_SWORDS` in a PZ. `ICON_PIGEON = 1 << 14`
(`const.h:343`). Icons are sent via the `0xA2` packet (`sendIcons`).

### Current behaviour

`enter_world::icons()` writes a static `[0xA2][u16 0]` (no icons), once in the
enter-world burst.

### New behaviour

- Parameterize the encoder: `enter_world::icons(mask: u16) -> Vec<u8>`
  (`[0xA2][u16 mask LE]`). The burst calls `icons(initial_mask)`.
- Track the player's PZ state. In `do_move`, after the position update, compute
  `is_protection_zone(new)`; if it differs from `is_protection_zone(old)`, push
  `icons(mask)` to the mover with the `ICON_PIGEON` (`0x4000`) bit set or cleared.
- Initial burst: if the spawn tile is a PZ, send the bit; else `0`.
- M7.1 only manages the `ICON_PIGEON` bit; other icon bits stay `0` until their
  features (conditions, in-fight swords) land.

## 3. Blood-hit effect fix

### TFS reference

On a hit to a blood creature, TFS emits `CONST_ME_DRAWBLOOD` (`game.cpp:3887`).
Our M5 magic-effect convention is **wire = TFS enum − 1** (e.g.
`EFFECT_TELEPORT = 10` for `CONST_ME_TELEPORT = 11`, verified working live).

### Current behaviour

`apply_damage` already pushes the effect to the victim **and** all spectators
(`game.rs`). `EFFECT_DRAWBLOOD = 0` (`enter_world.rs:26`). A wire value of `0`
likely renders nothing on the client (no/null effect).

### Action

Verify the correct wire value for the physical-hit blood effect against the
OTClient effect table and fix `EFFECT_DRAWBLOOD` accordingly (the `−1`
convention may not hold at the bottom of the enum, or `0` is reserved). This is a
constant fix + a live confirmation; no logic change. The byte-faithful tests
already cover `magic_effect`'s shape.

## Tests

- **Death:** victim receives `0x28`; death-tile spectators receive the id-form
  remove; victim is removed from the `players` map; no in-world re-add. (Replaces
  the old respawn-invariant tests.)
- **Relog after death:** a fresh `login()` spawns at the temple with full HP.
- **PZ badge:** move into a PZ → mover gets `0xA2` with `0x4000`; move out →
  `0xA2` with `0`; spawn in a PZ → burst carries the bit.
- **Blood effect:** `magic_effect` byte shape unchanged; the corrected
  `EFFECT_DRAWBLOOD` value is asserted.

Gate (as M7): `cargo test` green, `cargo clippy --all-targets -- -D warnings`
clean, `#![forbid(unsafe_code)]` intact.

## Live acceptance

Real OTClient: (1) die in PvP → death window shows → returns to character select
→ relog spawns at the temple with full HP; (2) standing in the temple shows the
PZ dove badge, leaving it clears the badge; (3) hits draw a visible blood
animation on the struck creature.
