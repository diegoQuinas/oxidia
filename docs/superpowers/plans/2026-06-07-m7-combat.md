# M7 — Combat Core & PvP Melee — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Two players can fist-fight: target (`0xA1`), auto-swing on a timer, TFS skill-based melee damage, health-bar (`0x8C`) + self-stats (`0xA0`) sync, death window (`0x28`), and respawn to the temple — with protection-zone attack rejection (`0xB4`).

**Architecture:** The single authoritative actor (`world::game::GameWorld`) gains **combat state** (`attacking`, `last_attack_ms`, `health`) on `PlayerState` and a **single global combat-tick** (`tokio::time::interval` → `Command::CombatTick`) so server-driven swings stay serialized through the one writer — no per-fight timers, no locks (the M5 model). Damage math is a **pure feeder** (`world/src/combat.rs`). Wire packets are a **pure feeder** (`protocol/src/combat_packets.rs`); the victim's own `0xA0` reuses the existing `enter_world::stats`.

**Tech Stack:** Rust, tokio (mpsc, `time::interval`), `rand` for the damage spread, the existing `protocol` byte-faithful encoders + M5 spectator/push machinery. `#![forbid(unsafe_code)]`, strict clippy, TDD.

**Parallelization:** Per `docs/superpowers/plans/2026-06-06-m5-m9-worktree-parallelization.md` — feeders (wt-data, wt-proto) build in isolated worktrees and merge to `main` ahead of the spine; **only wt-spine edits `game.rs`, `game_service.rs`, `map.rs`, `PROGRESS.md`**. Tasks below are tagged with their target worktree. Feeder tasks (1, 2) are independent and may run in parallel; spine tasks (3–6) are serial and depend on 1+2 being merged.

Design reference: `docs/superpowers/specs/2026-06-07-m7-combat-design.md`. TFS evidence cited inline.

**Wire-format facts (verified in `reference/tfs/`):**

- **Inbound attack `0xA1`** (`protocolgame.cpp:537`, `parseAttack` 972-977): `[0xA1][u32 creatureId]`; `id == 0` clears the target.
- **Inbound follow `0xA2`** (`parseFollow` 979-984): `[0xA2][u32 creatureId]` — consumed and ignored in M7.
- **`0x8C` creature health** (`sendCreatureHealth` 2339-2351): `[0x8C][u32 id][u8 percent]`, `percent = ceil(hp / max(maxHp,1) * 100)`.
- **`0xA0` self-stats** (`AddPlayerStats` 3007-3048): existing `enter_world::stats` (53-byte body, `health`/`maxHealth` as the leading `u16`s) — reused, not rebuilt.
- **`0x28` death/relogin window** (`sendReLoginWindow` 1376-1383): `[0x28][u8 0x00][u8 unfairFightReduction]`; M7 sends `0`.
- **`0xB4` text message** (`sendTextMessage` 1411): the PZ-rejection status line.
- **Melee max damage** (`getMaxWeaponDamage` weapons.cpp:135-138): `round((level/5) + ((((skill/4.)+1)*(value/3.))*1.03)/factor)`; **`level/5` is integer, `skill/4.` and `value/3.` are float.**
- **Fist constants** (`useFist`): `attackValue = 7`, `attackSkill = SKILL_FIST`, `attackFactor = 1.0` (attack mode); damage `= -uniform_random(0, max)`.
- **Attack interval** (`getAttackSpeed` 351-358, `doAttacking` 3223-3264): no-vocation default `2000 ms`.
- **PZ flag** (`iomap.h:60` `OTBM_TILEFLAG_PROTECTIONZONE = 1<<0`, parsed into `MapTile.flags`; rejected in `combat.cpp:294-297`).

#![forbid(unsafe_code)]
//! Pure melee combat math — a byte-faithful port of TFS 1.4.2
//! `Weapons::getMaxWeaponDamage` (weapons.cpp:135-138) and `Weapon::useFist`.
//! No actor, no protocol, no I/O. Parameterized by every input so M10 (equipped
//! weapons) and M14 (real skills/levels/vocation) plug in without changes here.

use rand::Rng;

/// Fist (unarmed) attack value — TFS `useFist` constant.
pub const FIST_ATTACK_VALUE: i32 = 7;
/// Attack-mode factor (FIGHTMODE_ATTACK) — `Player::getAttackFactor`.
pub const ATTACK_FACTOR: f64 = 1.0;

/// `round((level/5) + ((((skill/4.)+1)*(value/3.))*1.03)/factor)`.
/// CRITICAL: `level / 5` is INTEGER division; `skill/4.` and `value/3.` are
/// FLOAT division — mirrors the C++ exactly (a float `level/5` inflates damage).
pub fn max_weapon_damage(level: u32, attack_skill: i32, attack_value: i32, attack_factor: f64) -> i32 {
    let int_part = (level / 5) as f64;
    let skill_term = ((attack_skill as f64 / 4.0) + 1.0) * (attack_value as f64 / 3.0);
    (int_part + (skill_term * 1.03) / attack_factor).round() as i32
}

/// Uniform damage in `0..=max_weapon_damage(..)` (TFS melee `normal_random` is a
/// uniform int distribution). Returns the POSITIVE magnitude.
pub fn melee_damage(rng: &mut impl Rng, level: u32, attack_skill: i32, attack_value: i32, attack_factor: f64) -> i32 {
    let max = max_weapon_damage(level, attack_skill, attack_value, attack_factor).max(0);
    if max == 0 { 0 } else { rng.gen_range(0..=max) }
}

/// Fist swing: `attack_value = 7`, `attack_factor = 1.0`.
pub fn fist_damage(rng: &mut impl Rng, level: u32, fist_skill: i32) -> i32 {
    melee_damage(rng, level, fist_skill, FIST_ATTACK_VALUE, ATTACK_FACTOR)
}

#[cfg(test)]
mod tests {
    use super::*;
    use rand::{rngs::StdRng, SeedableRng};

    #[test]
    fn level1_fist_max_is_eight() {
        // round((1/5)=0 + ((((10/4.)+1)*(7/3.))*1.03)/1.0) = round(8.408..) = 8
        assert_eq!(max_weapon_damage(1, 10, 7, 1.0), 8);
    }

    #[test]
    fn integer_level_division_floors() {
        // level 4/5 == 0 (integer), level 5/5 == 1.
        assert_eq!(max_weapon_damage(4, 0, 0, 1.0), 0);
        assert_eq!(max_weapon_damage(5, 0, 0, 1.0), 1);
    }

    #[test]
    fn fist_damage_within_bounds() {
        let mut rng = StdRng::seed_from_u64(42);
        let max = max_weapon_damage(1, 10, 7, 1.0);
        for _ in 0..1000 {
            let d = fist_damage(&mut rng, 1, 10);
            assert!((0..=max).contains(&d), "{d} out of 0..={max}");
        }
    }
}
```

//! Combat wire packets for protocol 10.98. Byte-faithful ports of
//! `reference/tfs/src/protocolgame.cpp`: `sendCreatureHealth` (2339-2351, 0x8C),
//! `sendReLoginWindow` (1376-1383, 0x28), and the inbound `parseAttack`
//! (972-977, 0xA1). The victim's own 0xA0 self-stats is NOT here — it is reused
//! from `crate::enter_world::stats`.

use crate::message::MessageWriter;

pub const OP_ATTACK: u8 = 0xA1;          // inbound: [0xA1][u32 creatureId]
pub const OP_CREATURE_HEALTH: u8 = 0x8C; // outbound: [0x8C][u32 id][u8 percent]
pub const OP_DEATH_WINDOW: u8 = 0x28;    // outbound: [0x28][0x00][u8 unfairFight]

/// ceil(health / max(maxHealth,1) * 100), clamped to 0..=100 — the TFS percent.
pub fn health_percent(health: i32, max_health: i32) -> u8 {
    let max = max_health.max(1);
    let pct = ((health.max(0) as f64 / max as f64) * 100.0).ceil() as i32;
    pct.clamp(0, 100) as u8
}

/// `0x8C` creature health-bar percent.
pub fn creature_health(creature_id: u32, percent: u8) -> Vec<u8> {
    let mut w = MessageWriter::new();
    w.write_u8(OP_CREATURE_HEALTH);
    w.write_u32(creature_id);
    w.write_u8(percent);
    w.into_bytes()
}

/// `0x28` death / relogin window.
pub fn death_window(unfair_fight_reduction: u8) -> Vec<u8> {
    vec![OP_DEATH_WINDOW, 0x00, unfair_fight_reduction]
}

/// Parse an inbound `0xA1` attack body (the bytes AFTER the opcode):
/// `[u32 creatureId]`. `Some(0)` means "clear target". `None` if malformed.
pub fn parse_attack(body: &[u8]) -> Option<u32> {
    let b: [u8; 4] = body.get(..4)?.try_into().ok()?;
    Some(u32::from_le_bytes(b))
}

#[cfg(test)]
mod tests {
    use super::*;

    #[test]
    fn health_percent_ceils_and_clamps() {
        assert_eq!(health_percent(150, 150), 100);
        assert_eq!(health_percent(0, 150), 0);
        assert_eq!(health_percent(1, 150), 1);   // ceil(0.66) = 1
        assert_eq!(health_percent(149, 150), 100); // ceil(99.3) = 100
        assert_eq!(health_percent(5, 0), 100);   // max(0,1) guard
    }

    #[test]
    fn creature_health_layout() {
        let p = creature_health(0x1000_0001, 73);
        assert_eq!(p[0], OP_CREATURE_HEALTH);
        assert_eq!(u32::from_le_bytes([p[1], p[2], p[3], p[4]]), 0x1000_0001);
        assert_eq!(p[5], 73);
        assert_eq!(p.len(), 6);
    }

    #[test]
    fn death_window_layout() {
        assert_eq!(death_window(0), [OP_DEATH_WINDOW, 0x00, 0x00]);
    }

    #[test]
    fn parse_attack_reads_le_id_and_clear() {
        let body = 0x1000_0002u32.to_le_bytes();
        assert_eq!(parse_attack(&body), Some(0x1000_0002));
        assert_eq!(parse_attack(&0u32.to_le_bytes()), Some(0)); // clear
        assert_eq!(parse_attack(&[1, 2]), None); // short
    }
}
```

tokio::spawn(async move {
