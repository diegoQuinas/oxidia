# M4 — Core Walk + Visible Creature Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Render the player creature with its outfit and let it walk one floor of the Thais temple with collision, over the real OTClient Redemption.

**Architecture:** Pure request/response over the existing `world` actor. The connection task parses incoming walk opcodes, asks the actor to move/turn the player, then encodes and sends `0x6D` + map slices (or `sendCancelWalk`) back. No per-player back-channel — a single player has no spectators. The `0x64` map description is extended to serialize the player creature on its tile (byte-faithful port of TFS `AddCreature` for 1098), which is what fixes the currently-invisible knight.

**Tech Stack:** Rust (edition 2024, `#![forbid(unsafe_code)]`), tokio actor over mpsc/oneshot, TFS 1.4.2 as the byte-layout oracle (`reference/tfs/`).

**Spec:** `docs/superpowers/specs/2026-06-06-m4-walk-design.md`

## Key facts established from TFS (do not re-derive)

- **Tile thing order** (`protocolgame.cpp:583` `GetTileDescription`): `[u16 0x0000 env]`, ground item, top items, **creatures (reverse order)**, down items. The tile ends when the next `[skip][0xFF]` flush appears — there is **no** per-tile terminator.
- **Every item is `[u16 clientId][u8 0xFF]`** at 1098 (`networkmessage.cpp:86-88` — the `0xFF` is `MARK_UNMARKED`, part of the item). The existing `add_item` is already correct; creatures slot in **after** the ground item's two-byte mark.
- **Creature markers** `0x0061` (unknown) / `0x0062` (known) are `< 0xFF00`, so the client parses them as tile things, not skip markers.
- **Direction wire bytes** (`position.h`): N=0, E=1, S=2, W=3, SW=4, SE=5, NW=6, NE=7.
- **`addPosition`** = `[u16 x][u16 y][u8 z]` (5 bytes).
- **`0x6D` creature move** (`protocolgame.cpp:2588`, stackpos<10): `[0x6D][oldPos 5][u8 oldStackPos][newPos 5]`. A lone creature on a ground-only tile has **stackpos = 1**.
- **Slice arithmetic** (`protocolgame.cpp:2616-2630`, `maxClientViewportX=8`, `maxClientViewportY=6`):
  - north (`oldPos.y>newPos.y`): `0x65` + `GetMapDescription(oldPos.x-8, newPos.y-6, newPos.z, 18, 1)`
  - south (`oldPos.y<newPos.y`): `0x67` + `GetMapDescription(oldPos.x-8, newPos.y+7, newPos.z, 18, 1)`
  - east (`oldPos.x<newPos.x`): `0x66` + `GetMapDescription(newPos.x+9, newPos.y-6, newPos.z, 1, 14)`
  - west (`oldPos.x>newPos.x`): `0x68` + `GetMapDescription(newPos.x-8, newPos.y-6, newPos.z, 1, 14)`
  - **Diagonal**: the y and x blocks are independent `if`s — a diagonal emits `0x6D` then **both** slices (NOT a full `0x64`).
- **`GetMapDescription(x,y,z,width,height)`** loops floors 7→0 with `offset = z - nz`, `getTile(x+nx+offset, y+ny+offset, nz)`. Slices use the same function (full z range, just `width×height` of 18×1 or 1×14).
- **`sendCancelWalk`** (`protocolgame.cpp:2287`): `[0xB5][u8 direction]`.
- **`sendCreatureTurn`** (`protocolgame.cpp:2176`, stackpos<10): `[0x6B][pos 5][u8 stackPos][u16 0x0063][u32 id][u8 direction][u8 walkthrough]`.
- **`AddCreature`** (`protocolgame.cpp:2935`, no per-field version gates at 1097–1098). Unknown path fields in order: `u16 0x0061`, `u32 removeId`, `u32 id`, `u8 creatureType`, `string name`, `u8 health%`, `u8 direction`, **outfit**, `u8 light.level`, `u8 light.color`, `u16 stepSpeed/2`, `u8 skull`, `u8 partyShield`, `u8 guildEmblem (unknown only)`, `u8 creatureType2`, `u8 speechBubble`, `u8 mark=0xFF`, `u16 helpers`, `u8 walkthrough`. Known path replaces the first three fields with `u16 0x0062` + `u32 id` and omits `guildEmblem`.
- **`AddOutfit`** (`protocolgame.cpp:3066`): `u16 lookType`; if `!=0` → `u8 head,body,legs,feet,addons`; else → `u16 lookTypeEx`; then always `u16 mount`.
- **`items.otb` block-solid flag** (`items.cpp`): `FLAG_BLOCK_SOLID = 1 << 0` (bit 0 of the per-item `u32 flags`). `ItemType.flags` is already parsed by the `formats` crate.

```rust
impl Position {
    /// Translate by `(dx, dy)` on the same floor. Returns `None` if it would
    /// leave the `u16` coordinate range.
    pub fn offset(self, dx: i32, dy: i32) -> Option<Position> {
        let x = i32::from(self.x) + dx;
        let y = i32::from(self.y) + dy;
        if (0..=i32::from(u16::MAX)).contains(&x) && (0..=i32::from(u16::MAX)).contains(&y) {
            Some(Position::new(x as u16, y as u16, self.z))
        } else {
            None
        }
    }
}

/// A facing/movement direction. Wire bytes match TFS `Direction` (`position.h`).
#[derive(Debug, Clone, Copy, PartialEq, Eq)]
pub enum Direction {
    North,
    East,
    South,
    West,
    NorthEast,
    SouthEast,
    SouthWest,
    NorthWest,
}

impl Direction {
    /// The protocol byte the client expects (N=0, E=1, S=2, W=3, SW=4, SE=5, NW=6, NE=7).
    pub const fn to_byte(self) -> u8 {
        match self {
            Direction::North => 0,
            Direction::East => 1,
            Direction::South => 2,
            Direction::West => 3,
            Direction::SouthWest => 4,
            Direction::SouthEast => 5,
            Direction::NorthWest => 6,
            Direction::NorthEast => 7,
        }
    }

    /// `(dx, dy)` step on the tile grid for this direction.
    pub const fn delta(self) -> (i32, i32) {
        match self {
            Direction::North => (0, -1),
            Direction::East => (1, 0),
            Direction::South => (0, 1),
            Direction::West => (-1, 0),
            Direction::NorthEast => (1, -1),
            Direction::SouthEast => (1, 1),
            Direction::SouthWest => (-1, 1),
            Direction::NorthWest => (-1, -1),
        }
    }
}
```

```rust
/// Exact port of TFS `GetMapDescription` + `GetFloorDescription`
/// (protocolgame.cpp:633-680). Floors 7->0, `skip` persists across them and
/// starts at -1. After a tile's ground item, any creature on that tile is spliced
/// in (TFS writes creatures as tile things after the items).
#[allow(clippy::too_many_arguments)]
fn get_map_description<S: GroundSource>(
    w: &mut MessageWriter,
    anchor_x: i32,
    anchor_y: i32,
    center_z: i32,
    width: i32,
    height: i32,
    src: &S,
    creatures: &[PlacedCreature],
) {
    let mut skip: i32 = -1;
    for nz in (0..=7i32).rev() {
        let offset = center_z - nz;
        for nx in 0..width {
            for ny in 0..height {
                let wx = anchor_x + nx + offset;
                let wy = anchor_y + ny + offset;
                match src.ground(wx, wy, nz) {
                    Some(client_id) => {
                        if skip >= 0 {
                            w.write_u8(skip as u8);
                            w.write_u8(0xFF);
                        }
                        skip = 0;
                        w.write_u16(0x0000); // environmental effects placeholder
                        add_item(w, client_id);
                        for c in creatures {
                            if i32::from(c.x) == wx && i32::from(c.y) == wy && i32::from(c.z) == nz {
                                w.write_bytes(&c.bytes);
                            }
                        }
                    }
                    None => {
                        if skip == 0xFE {
                            w.write_u8(0xFF);
                            w.write_u8(0xFF);
                            skip = -1;
                        } else {
                            skip += 1;
                        }
                    }
                }
            }
        }
    }
    if skip >= 0 {
        w.write_u8(skip as u8);
        w.write_u8(0xFF);
    }
}
```

```rust
/// move, then the newly-revealed row/column slice(s). Ports the independent y/x
/// `if` blocks of TFS `sendMoveCreature` (2616-2630) — a diagonal emits both.
/// Stackpos is fixed at 1 (a lone player creature on a ground-only tile).
pub fn walk_update<S: GroundSource>(
    old: (u16, u16, u8),
    new: (u16, u16, u8),
    src: &S,
    creatures: &[PlacedCreature],
) -> Vec<u8> {
    let mut out = creature_move(old, 1, new);
    let (ox, oy) = (i32::from(old.0), i32::from(old.1));
    let (nx, ny) = (i32::from(new.0), i32::from(new.1));
    let nz = i32::from(new.2);

    if oy > ny {
        out.extend(map_description::encode_slice(
            SLICE_NORTH, ox - VIEW_X, ny - VIEW_Y, nz, SLICE_W, 1, src, creatures,
        ));
    } else if oy < ny {
        out.extend(map_description::encode_slice(
            SLICE_SOUTH, ox - VIEW_X, ny + (VIEW_Y + 1), nz, SLICE_W, 1, src, creatures,
        ));
    }
    if ox < nx {
        out.extend(map_description::encode_slice(
            SLICE_EAST, nx + (VIEW_X + 1), ny - VIEW_Y, nz, 1, SLICE_H, src, creatures,
        ));
    } else if ox > nx {
        out.extend(map_description::encode_slice(
            SLICE_WEST, nx - VIEW_X, ny - VIEW_Y, nz, 1, SLICE_H, src, creatures,
        ));
    }
    out
}
```

```rust
async fn run_session<S>(

    world: &WorldHandle,

                    Err(e) => {
                        tracing::debug!(?e, "dropping undecryptable frame");
                        continue;
                    }
                };
                if let Some(&opcode) = payload.first() {
                    handle_client_packet(stream, keys, world, &mut session, opcode).await?;
                }
            }
            Err(_elapsed) => {
                send_encrypted(stream, keys, &[PING_OPCODE]).await?;
            }
        }
    }
    Ok(())
}

/// Map an incoming opcode to a (direction, is_turn) action, or `None` to drain.
fn opcode_action(opcode: u8) -> Option<(Direction, bool)> {
    match opcode {
        0x65 => Some((Direction::North, false)),
        0x66 => Some((Direction::East, false)),
        0x67 => Some((Direction::South, false)),
        0x68 => Some((Direction::West, false)),
        0x6A => Some((Direction::NorthEast, false)),
        0x6B => Some((Direction::SouthEast, false)),
        0x6C => Some((Direction::SouthWest, false)),
        0x6D => Some((Direction::NorthWest, false)),
        0x6F => Some((Direction::North, true)),
        0x70 => Some((Direction::East, true)),
        0x71 => Some((Direction::South, true)),
        0x72 => Some((Direction::West, true)),
        _ => None, // pong (0x1E), logout handled by EOF, unknown -> drain
    }
}

async fn handle_client_packet<S>(
    stream: &mut S,
    keys: &xtea::RoundKeys,
    world: &WorldHandle,
    session: &mut Session,
    opcode: u8,
) -> Result<()>
where
    S: AsyncRead + AsyncWrite + Unpin,
{
    let Some((direction, is_turn)) = opcode_action(opcode) else {
        return Ok(());
    };

    if is_turn {
        if let Some(facing) = world.turn_player(session.id, direction).await {
            session.facing = facing;
            let pos = (session.pos.x, session.pos.y, session.pos.z);
            let pkt = walk::creature_turn(pos, 1, session.id, facing.to_byte());
            send_encrypted(stream, keys, &pkt).await?;
        }
        return Ok(());
    }

    if let Some(res) = world.move_player(session.id, direction).await {
        session.facing = res.facing;
        match res.outcome {
            MoveOutcome::Moved { from, to } => {
                session.pos = to;
                let pkt = walk::walk_update(
                    (from.x, from.y, from.z),
                    (to.x, to.y, to.z),
                    world.map.as_ref(),
                    &[],
                );
                send_encrypted(stream, keys, &pkt).await?;
            }
            MoveOutcome::Blocked => {
                let pkt = walk::cancel_walk(res.facing.to_byte());
                send_encrypted(stream, keys, &pkt).await?;
            }
        }
    }
    Ok(())
}
```
