# M7.1 Combat Polish Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Make M7 death behave like TFS (death → logout → relog at temple), show the protection-zone client badge, and render the blood-hit effect.

**Architecture:** Three small, mostly independent changes. Death becomes a logout (drop the victim from the actor's `players` map → the session's `push_tx` drops → the writer channel closes → the existing `select!` tears the session down; no `game_service.rs` change). The PZ badge is the `0xA2` icons packet re-sent on protection-zone boundary crossings. The blood effect is a one-constant wire-value fix.

**Tech Stack:** Rust (edition 2024), tokio actor (`world::game`), byte-faithful protocol encoders (`protocol`). Test runner: `cargo test`. Lint gate: `cargo clippy --all-targets -- -D warnings`.

Spec: `docs/superpowers/specs/2026-06-07-m7.1-combat-polish-design.md`.

```rust
    /// Handle the death of `victim_id`: push the `0x28` death window, clear all
    /// fights targeting the victim, remove the creature from the world (id-form
    /// remove to spectators + drop from their known-sets), and remove the victim
    /// from the `players` map. Dropping the victim's `PlayerState` drops its
    /// session `push_tx`, which closes the writer channel and tears the session
    /// down — the client shows the death window and returns to character select.
    /// Respawn at the temple happens on the next login (`login` uses `free_spawn`).
    /// Mirrors TFS `onDeath` -> `sendReLoginWindow` + `removeCreature`
    /// (player.cpp:2070, 2197).
    fn do_death(&mut self, victim_id: u32) {
        let death_pos = match self.players.get(&victim_id) {
            Some(p) => p.position,
            None => return,
        };

        // Stop every attacker locked on the victim.
        for p in self.players.values_mut() {
            if p.attacking == Some(victim_id) {
                p.attacking = None;
            }
        }

        // Death window to the victim.
        self.push(victim_id, combat_packets::death_window(0));

        // Remove from the death tile for every spectator. The id-form remove is
        // unambiguous under co-occupancy (stair/height landings); also drop the
        // victim from each spectator's known-set so a re-login re-introduces it
        // with the full 0x61 form.
        let spectators_death: Vec<u32> = self.spectators(death_pos, victim_id);
        for spec in &spectators_death {
            self.push(*spec, walk::remove_creature_by_id(victim_id));
            if let Some(s) = self.players.get_mut(spec) { s.known.remove(&victim_id); }
        }

        // Remove the victim from the world. This drops its push_tx -> the writer
        // channel closes -> the session ends -> the client returns to char select.
        self.players.remove(&victim_id);
    }
```
